# Constraints
- BLOCKING:
  - reviewer split where total prompt size does not shrink AND concerns-per-unit does not drop (see `.opencode/agent/_iterate/rules/split-decision-rule.txt`)
  - reviewer split where one reviewer needs `changed_paths` only and another needs wider reads (callers, error collection, repo scan); different file scopes, different reviewers
  - pipeline phase kept as a subagent when it has no LLM judgement
  - high-risk ownership lost in a reviewer merge
  - numbered action-file runtime contracts or cacheless/inline reviewers that create/read action sidecars
  - copied boilerplate where a renderer import would do
- ADVISORY: minor template-style drift, missing block-form import for multi-arg includes, or repo-relative path convention drift.
- Keep response compact; detailed evidence belongs in cache (cached mode only).
