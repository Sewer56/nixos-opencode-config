<prompt_optimization_contract>
Scope: changed model-facing instructions: command bodies, agent/reviewer prompts, prompt templates, prompt docs.

<boundaries>
- Prompt text owns goal, scope, inputs, source boundaries, task-level tool behavior, output contract, stop/fallback rules, verification.
- Harness/config owns model choice, reasoning/effort params, tool schemas/MCP wiring, provider reasoning replay, permissions, sandbox/egress, cache mechanics.
- If target is harness/config docs, keep harness advice there; do not copy it into prompts.
</boundaries>

<shape>
- Start with deliverable and done criteria.
- Include exact inputs, editable scope, non-goals, output schema, failure behavior, and verification evidence.
- Use XML-style tags when one prompt mixes instructions, context, examples, source data, or schemas. Keep simple prompts flat.
- Use [[placeholder_name]] for placeholder text inside examples/schemas; reserve angle brackets for real XML tags.
</shape>

<density>
- Cut persona fluff, pleasantries, stale model hacks, generic rationale, duplicated caller/callee rules, copied catalogs, and examples that do not constrain behavior.
- Keep safety/source boundaries, permissions implied by frontmatter, schemas, stop rules, and checks.
- Prefer positive scoped rules and if/then branches over broad always/never lists.
</density>

<context_and_agents>
- Read target paths and direct references before changing repo-fact claims.
- Bound discovery by target, convergence, and validation failures; do not add open-ended thoroughness.
- Use subagents/reviewers only when separate context or fresh judgment reduces main context cost. Handoff = paths/ids/flags/criteria/artifact paths, not parent workflow or catalogs.
</context_and_agents>

<verification>
- Define smallest useful check: render/static/test/lint/build/smoke or inspectable substitute.
- Record not-run checks with reason and next-best evidence.
- Reject success claims based only on appearance or intent.
</verification>

<review_failures>
BLOCKING: missing output contract, missing verification, prompt/harness mixing, placeholder/XML ambiguity, hidden-reasoning transcript request, untrusted source treated as instruction, selected PE/OPT/WOPT carry-in absent, over-compression that removes required boundary/schema/check.
ADVISORY: local wording economy, harmless doc drift, redundant example with no behavior risk.
</review_failures>
</prompt_optimization_contract>
