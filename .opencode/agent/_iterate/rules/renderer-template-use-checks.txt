<renderer_template_checks>
- Prefer renderer imports/args over copied boilerplate.
- Use repo-relative paths. For `.opencode/` files importing from `config/`, use `../config/`.
- Use block imports for multi-arg templates.
- Keep `{{arg:name}}` and conditionals inside reusable templates unless editing such template.
- Forward nested include args explicitly.
- Use env tokens only for true runtime environment values.
- Use `{{path:./...}}` for scripts/binaries needing absolute path at render time.
- Use `[[name]]` for human placeholders; reserve angle brackets for real XML tags.
- Omit false flags; use `1` for true.
- Do not pass args target template does not consume.
- Validate changed command/agent prompts with `bash {{path:./scripts/render-file.sh}} [[repo_relative_path]]` when renderer syntax changed.

```text
Read /absolute/path/to/config/rules/groups/tests/target-test-strategy.md and follow it.
```

Good:
```markdown
{{ file="./rules/groups/tests/target-test-strategy.md" }}
```
