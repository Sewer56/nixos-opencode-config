# Split Rule

Apply on add/split/merge/remove of subagent, reviewer, pipeline phase.

## Why

Long prompt lowers adherence. Narrow unit or script beats fat agent.

## Split when

- Runner exceeds 150 lines or holds 3+ heterogeneous domains.
- Reviewer emits findings across unrelated concerns.
- Phase has no LLM judgment and returns `PASS | BLOCKING`.
- Caller repeats callee scope, role, or schema. Move it to callee.
- Reviewers need different file scopes: one needs `changed_paths`; another needs callers, errors, or repo scan.

## Keep when

- Child size matches parent and parent already narrow (<3 domains). Redistribution, no win.
- Units read same artifacts and emit overlapping findings. Merge.
- Handoff contract longer than work saved.
- Phase is one or two sentences. Parent runs it cheaply.

## Metrics

1. Total prompt size: parent + children.
2. Concerns per unit: heterogeneous domains model holds at once.

Split wins when total grows but concerns drop, e.g. 9→2. Split loses when total holds and concerns stay. Measure concerns, not file length.

## Script over subagent

No LLM judgment + deterministic verdict + once per run = `scripts/<name>.sh`.

## Cost guard

New unit adds permission grant, sync file, handoff contract, edit set. If cost line exceeds removed work, keep parent.

## Anti-patterns

- Prep subagent as long as parent would be. Phase too small.
- Reviewer split into same-size prompts with same concerns. No gain.
- Subagent reads `bash` and returns `PASS | BLOCKING`. Use script.
- Caller re-lists callee `Scope:` or schema. Move to callee; drop caller copy.

## After

- Update runner `task:` allowlist or `bash:` grant.
- Update iterate doc and reviewer `cache_path` / artifact names.
- Verify parent shorter, child narrower. Else revert.
