
<split_decision_rule>
Apply when adding, splitting, merging, or removing subagent/reviewer/pipeline phase.

<split_when>
- Broad investigation would pollute primary context but can return compact path/evidence summary.
- Fresh-context review can judge diff against requirements without writer bias.
- One unit holds 3+ heterogeneous concerns or >150 prompt lines.
- Review scopes need different artifacts: changed paths vs callers/errors/repo scan.
- Phase is deterministic PASS/BLOCKING: make script/hook, not subagent.
- Caller repeats callee role/process/schema: move to callee, pass smaller handoff.
</split_when>

<keep_or_merge_when>
- Child prompt/read scope equals parent; handoff longer than saved work.
- Units read same artifacts and emit overlapping findings.
- Phase is one or two simple sentences.
- Existing deterministic script/hook can decide.
</keep_or_merge_when>

<metrics>
Judge concern count, context footprint, handoff cost, permission cost, and total runtime prompt size. Split wins when concern/context load drops enough to pay handoff cost. File count alone is not evidence.
</metrics>

<handoff_contract>
Give subagent: goal, owned scope, exact paths/artifacts, criteria, output shape. Do not give copied parent workflow, full catalogs, unrelated permissions, or "inspect everything".
</handoff_contract>

<anti_patterns>
- Prep subagent as long as parent.
- Reviewers split by name but same scope.
- Subagent wraps shell/static check.
- Broad review asks for "anything improved" then creates style churn.
- Independent reviewers serialized without dependency.
</anti_patterns>
</split_decision_rule>
