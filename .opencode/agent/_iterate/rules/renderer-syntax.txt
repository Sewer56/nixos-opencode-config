# Renderer Template Syntax
Reference for OpenCode prompt imports, args, conditionals, env tokens. Contains live renderer tokens; exclude from broad interpolation validation.

## Imports
- File: `{{file="./path/to/file.md"}}`
- With args: `{{file="./template.txt" arg="value"}}`
- Prefer block import for multi-arg calls.

Good:
```markdown
{{ file="./rules/groups/tests/target-test-strategy.md" }}
```

Good parameterized:
```markdown
{{
  file="../config/agent/_templates/review-mission.txt"
  artifact_type="implementation"
  domain="implementation"
}}
```

## Template placeholders and args
- Reusable-template placeholder: `{{arg:name}}`.
- Human-readable placeholder inside examples/schemas: `[[name]]`; do not use angle placeholders.
- Args apply to one include. Forward args explicitly to nested includes.
- Boolean true: `1`. False: omit arg; do not pass `=0`.

## Conditionals
- Truthy: `{{if=name}} ... {{endif}}`
- Equality: `{{if=name==value}} ... {{endif}}`
- Inequality: `{{if=name!=value}} ... {{endif}}`
- Env: `{{if=env:CI}} ... {{else}} ... {{endif}}`

## Env and paths
- Runtime env value: `{{env:VAR}}`
- Script/binary absolute path: `{{path:./scripts/tool.py}}`
- Git-root absolute path: `{{gitpath:./scripts/tool.py}}`
- Do not use env tokens for local prompt/rule paths; use imports.

## Validation
- Do not pass unused args.
- Validate changed command/agent prompts with `bash {{path:./scripts/render-file.sh}} [[repo_relative_path]]` when imports, args, env tokens, or conditionals change.
