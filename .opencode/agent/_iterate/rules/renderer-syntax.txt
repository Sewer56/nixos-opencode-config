# Renderer Template Syntax

Usage guide for OpenCode prompt template imports, arguments, conditionals, and env tokens. This file intentionally contains live renderer tokens and is excluded from broad interpolation validation.

## File Imports

- Import a file: `{{file="./path/to/file.md"}}`
- Import with arguments: `{{file="./path/to/file.txt" arg1="value1" arg2="value2"}}`
- Use block imports for readability when passing multiple arguments.

Good:
```markdown
{{ file="./rules/groups/tests/target-test-strategy.md" }}
```

Good parameterized import:
```markdown
{{
  file="../config/agent/_templates/review-mission.txt"
  artifact_type="implementation"
  domain="implementation"
}}
```

## Template Placeholders

- Use only inside reusable template files: `{{arg:name}}`
- Args apply to one file include.
- Nested includes receive args only when forwarded as `key=value` pairs.

## Conditionals

- Truthy arg: `{{if=name}} ... {{endif}}`
- Equality: `{{if=name==value}} ... {{endif}}`
- Inequality: `{{if=name!=value}} ... {{endif}}`
- Env condition: `{{if=env:CI}} ... {{else}} ... {{endif}}`

## Env Tokens

- Runtime environment value: `{{env:VAR}}`
- Do not use env tokens for local prompt/rule paths; use file imports.

## Boolean Args

- Pass `1` for true.
- Omit false flags entirely; do not pass `=0`.

## Validation

- Do not pass args the target template does not consume.
- Validate changed agent and command prompts with `bash scripts/render-file.sh <repo-relative-path>` when imports, args, env tokens, or conditionals change.
