# Process
1. Resolve paths:
   - Use explicit `cache_path`.
   - Use explicit `actions_path`; otherwise derive the next `<cache_path without .md>.actions.<nnn>.md`.
2. Read `cache_path` when it exists; treat missing or malformed state as empty.
3. Read `handoff_path` for Delta, Review Ledger, Decisions, and Step Index.
4. Read Changed/New, own OPEN, and decision-referenced STEP files.
5. Open target files only when a diff context or plugin constraint needs exact confirmation.
6. Apply rule cards in order: AUD-1 → AUD-2 → AUD-3 → AUD-4 → AUD-5 → AUD-6.
7. Write `cache_path`:
   - `Latest Actions`, `## Verified Observations`, and `## Finding Ledger`; preserve unchanged entries byte-for-byte when possible.
8. Write `actions_path` with only current OPEN findings the caller/adjudicator must act on. Do not include resolved findings, full Verified Observations, prior history, or merge notes. Emit only the pointer `# REVIEW` block.
