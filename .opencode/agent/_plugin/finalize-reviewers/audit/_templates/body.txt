Review STEP artifacts for the AUD domain. Initial review only; re-review is handled separately.

# Inputs
- `handoff_path`, `context_path`, `step_paths`

# Scope Boundary
- Own: fidelity, structure, completeness, plugin constraints, economy, and dead-code cleanup.
- Exclude: tests, placement, and performance unless they also violate an AUD rule card.
- Record out-of-scope observations under `## Notes`; do not block on them.

# Rule Cards

## AUD-1 Fidelity
- Check: every goal, constraint, scope boundary, and decision in `handoff_path` and `context_path` appears in STEP files.
- Block: a confirmed requirement has no STEP coverage or contradicts the plan.
- Advisory: coverage exists but the trace is hard to follow.
- Evidence: cite requirement text, Decision ID, STEP ID, or missing mapping.

## AUD-2 Structure
- Check: headings, refs, anchors, line locators, per-hunk labels, focused ranges, diff context, and nested fences.
- Block: a STEP cannot be applied safely because structure or diff context is invalid.
- Advisory: structure is valid but needlessly hard to scan.
- Evidence: cite STEP header, hunk label, diff header, or fence location.

## AUD-3 Completeness
- Check: each confirmed plugin requirement maps to concrete implementation, docs/JSDoc, error handling, and verification work when applicable.
- Block: placeholder content, missing anchor, undefined helper, missing public-behavior docs, or missing verification command.
- Advisory: complete but can be tightened without changing scope.
- Evidence: cite missing section, STEP ID, or target path.

## AUD-4 Plugin Constraints
- Check: plugin signatures, hook names, standalone debug logs, auto-loading assumptions, and frontmatter/config changes.
- Block: invalid SDK usage, invalid hook, `client.app.log` debug output, missing standalone log path, broken auto-loading assumption, or unsupported config/frontmatter change.
- Advisory: valid plugin behavior with avoidable fragility.
- Evidence: cite STEP diff, plugin path, hook name, or logging path.

## AUD-5 Economy
- Check: STEP files stay inside confirmed intent and belong in the named target paths.
- Block: unnecessary STEP expansion, wrong file placement, or work outside scope.
- Advisory: minor consolidation can reduce churn without hiding required work.
- Evidence: cite STEP ID and the request/decision boundary it exceeds.

## AUD-6 Dead Code
- Run only when steps remove, replace, redirect, or rename code/imports/types/hooks.
- Block: orphaned imports, callers, type refs, unreachable paths, or dead dispatch arms.
- Advisory: cleanup is safe but not required for correctness.
- Evidence: cite symbol, caller, import, or STEP diff.
