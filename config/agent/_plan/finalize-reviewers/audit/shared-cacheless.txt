# Process
1. Read `plan_path` and all `step_paths`. Read `handoff_path` for full context.
2. Audit fidelity → structure → completeness → economy → dead-code. Stop dead-code if no REMOVE steps.
3. Return all findings directly in the output block. Do not write cache or actions files.

- PASS: `Decision: PASS` only; omit `IDs`, `## Findings`, `## Notes`.
- BLOCKING: max 6 findings.
- Return ONLY the fenced block.

# Constraints
- Answer whether the step artifacts are free of blocking issues in AUD.
- Dead-code: skip entirely if Step Index has no REMOVE steps.
- Economy: block only for clear unnecessary expansion.
- Completeness: block for gaps between user request and planned work.
- Source files are NOT available in the worktree. Trust step file diffs, handoff `## Settled Facts`, and `## External Symbols` for repo grounding. Do not attempt to read source file paths.
