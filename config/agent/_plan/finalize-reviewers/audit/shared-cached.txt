# Process
1. Load cache and derive actions path: Read `cache_path` if it exists; treat missing/malformed as empty. Use its `Latest Actions` field and finding ledger for grounding. Derive `actions_path` when absent by globbing existing `<cache_path without .md>.actions.*.md` files and choosing the next three-digit `<nnn>` path, starting `001`. Read `handoff_path` for Delta, Review Ledger, Decisions, Step Index.
2. Read `plan_path` and selected `step_paths`. Trust step file diffs and handoff `## Settled Facts` for repo grounding. Only open source files for specific verification when a diff context line does not match or a finding needs exact confirmation.
3. Audit fidelity → structure → completeness → economy → dead-code. Stop dead-code if no REMOVE steps.
4. Write `cache_path`:
   - `Latest Actions: <actions_path>`, `## Verified Observations`, and `## Finding Ledger`.
5. Write `actions_path` with only current OPEN findings the caller/adjudicator must act on. Full history and verified observations stay in `cache_path`.
6. Emit only the pointer `# REVIEW` block.

# Constraints
- BLOCKING: max 6 findings. Cache findings in `cache_path`.
- Dead-code: skip entirely if Step Index has no REMOVE steps.
- Economy: block only for clear unnecessary expansion.
- Completeness: block for gaps between user request and planned work.
- Verified observations MUST include grounding snapshots.
- Source files are NOT available in the worktree. Trust step file diffs, handoff `## Settled Facts`, and `## External Symbols` for repo grounding. Do not attempt to read source file paths.
