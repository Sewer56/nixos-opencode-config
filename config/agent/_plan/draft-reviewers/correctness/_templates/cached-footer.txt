# Actions file format
```markdown
# Review Actions: correctness
Source Cache: <cache_path>

## Findings
### [COR-001]
Category: FIDELITY | ACTION | FILE_PATH | TEMPLATE_STRUCTURE | DIFF_HEADERS | ILLUSTRATIVE_SNIPPETS
Severity: BLOCKING | ADVISORY
Evidence: <section, [P#], path:line, diff header, or missing element>
Problem: <one line>
Fix: <smallest concrete correction>
~~~diff
<artifact_base>.draft.md
--- a/<artifact_base>.draft.md
+++ b/<artifact_base>.draft.md
 unchanged context
-incorrect content
+correct content
 unchanged context
~~~
```

# Output

```text
# REVIEW
Agent: correctness
Decision: PASS | ADVISORY | BLOCKING
Domains: COR
IDs: COR-001, COR-002, ...
```
- PASS keeps `Agent:`, `Decision: PASS`, and `Domains: COR`; omit `IDs`.
- Every ID in the response MUST exist in `actions_path`.

Return ONLY the block above — no introduction, no summary, no conversational wrapper, no text before `# REVIEW` or after the final line.

# Constraints
- Block for missing required sections, invalid diff headers, or implementation-prescriptive snippets that should be illustrative.
- Do not block for minor wording when structure and snippet style are valid.
- Cite section names and specific `[P#]` items as evidence.
- Keep findings short and specific.
- In the cache, include a unified diff after every finding's `Fix:` field targeting `context_path` with the exact text replacement.
- Follow the `# Process` section for cache and skip handling.
