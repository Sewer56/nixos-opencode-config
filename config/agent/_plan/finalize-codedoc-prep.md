Read the finalized handoff and step files, validate preconditions, deepen discovery for code documentation gaps, and write a code-doc pipeline state file. The main `_plan/finalize-code-docs` agent reads the state and proceeds directly to the documentation pass and review loop.

# Inputs
- Derive `slug` from request context as a 2–3 word identifier. Derive `artifact_base` as `PROMPT-PLAN-<slug>`.
- `handoff_path`: `<artifact_base>.handoff.md`
- `discovery_path`: `artifact/<artifact_base>.repo-discovery.md`
- `step_pattern`: `<artifact_base>.step.*.md`

# Artifacts
- `artifact_base`: `PROMPT-PLAN-<slug>`
- `state_path`: `<artifact_base>.codedoc-state.md`

# Process

## 1. Validate preconditions
- Read `handoff_path`. If missing, return `Status: FAIL` and stop.
- Resolve exact I# and T# `step_paths` from the Step Index or by reading all files matching `step_pattern`.
- If zero step files exist, return `Status: FAIL` and stop.

## 2. Read discovery cache
- Read `discovery_path` if it exists. Treat as read-only.
- Mark cache as stale when `Artifact Base` mismatches `artifact_base` or facts contradict current step paths.

## 3. Deepen discovery for gaps
- Use I#/T# step diffs plus `discovery_path` before any new repo reads.
- For each gap (unclear public API status, documentation placement, reachable error variants, missing/stale cache evidence):
  - Use targeted local `glob`/`grep`/`read` scoped to the exact file, path, symbol, and named gap.
- Use `mcp-search` for external library or API documentation expectations first when needed.
- Read referenced target source files only when both the step diff and cache lack context.

## 4. Write pipeline state
- Overwrite `state_path` with the `# Code-Doc Pipeline State Format` below.

# Code-Doc Pipeline State Format

```markdown
# Code-Doc Pipeline State
Artifact Base: <artifact_base>
Generated By: _plan/finalize-codedoc-prep

## Resolved Paths
- handoff_path: <absolute path>
- discovery_path: <absolute path or N/A>
- step_paths:
  - <absolute path>
  - ...

## Discovery Gaps Filled
- <gap and resolution or none>

## Known Gaps Remaining
- <gap or none>
```

# Output

Return exactly:

```text
Status: SUCCESS | FAIL
State Path: <absolute state_path>
Step Count: <n>
Summary: <one-line summary>
```
